Backline Boost

A local macOS practice player for drummers, with Original, Drum Boost, and Drumless playback modes.

Quick Start

  1. Import supported local audio files with Import Track, Import Folder, or drag and drop from Finder or the Apple Music app.
  2. Play immediately: every imported track is available as Original right away.
  3. Backline Boost renders Drums and Drumless in the background, one track at a time.
  4. When a track's render finishes, Drum Boost and Drumless light up automatically.
  5. Practice in the Player and tune the drum boost live while you play.
Drum Boost is a live mix. Backline Boost does not create a fixed boosted full-song file. It renders Drums and Drumless once, then mixes them live using the saved drum boost value for that song.

User Manual

Original Plays the managed copy of the imported song.
Drum Boost Live-mixes Drums and Drumless with the saved per-song boost setting.
Drumless Plays the separated backing track without drums.

Backline Boost stores each song locally and reads embedded title, artist, album, and artwork metadata when available. It does not stream from Apple Music, Spotify, or other subscription libraries.

Library View

The Library is the home screen for imported tracks.

  • Import Track imports one or more supported audio files.
  • Import Folder imports supported top-level audio files in a folder.
  • Drag and drop works for supported local audio files, from Finder or directly from the Apple Music app.
  • Re-importing a file that is already in your library is skipped with an Already in library notice.
  • Single-click a rendered track to inspect it without stealing active playback.
  • Double-click a rendered track title to play it from 0:00.
  • Deleting a track asks for confirmation, then removes Backline Boost-managed source, artwork, and active render files for that track.

Rendering

Every import automatically queues a full-track separation; there is nothing to configure up front.

  • Imported tracks play as Original immediately while they wait to render.
  • Renders run one at a time in the background; the Library shows each track's stage or queue position.
  • One full-track separation produces durable Drums and Drumless assets.
  • Drum Boost live-mixes those two files, so the boost is adjustable in the Player at any time — raising drums also lowers the backing mix enough to keep the level steadier.
  • A failed render shows a retry button on the Library row and in the Player.
  • Renders resume automatically after quitting or relaunching the app.
  • The render output folder and quality are configurable in Preferences.

Player View

The Player is the full practice surface for one track.

  • Switch among Original, Drum Boost, and Drumless.
  • Use previous, seek back 15 seconds, play/pause, seek forward 15 seconds, and next.
  • Click or drag the progress bar to scrub.
  • Adjust the saved drum boost for the song.
  • Use pitch-preserving practice speed from 0.5x to 1.5x.
  • Double-click the speed slider to reset to 1.0x.
  • Loop the full song or set A/B section markers on the waveform timeline.

Mini-player

The Mini-player stays at the bottom of the app and controls active playback.

  • Click the track area to open the Player view.
  • Use transport buttons, 15-second seeking, repeat, shuffle, and scrub controls.
  • Click or drag the volume bar to adjust active playback volume.
  • Use the source tag to cycle the active playback source.
  • If the selected source is missing, Backline Boost falls back to Original.

Playlists

Playlists are saved practice queues.

  • Create, rename, populate, reorder, play, and delete playlists.
  • Deleting a playlist does not delete library tracks or audio assets.
  • Choose a playlist default source such as Drum Boost or Drumless.
  • Double-click a playlist item to start that track in the playlist queue.
  • Drum boost values are stored per song, not per playlist.

Preferences

Open Backline Boost Settings (⌘,). The window has three sections:

  • Playback — Normalize playback volume, a loudness assist that evens out big volume jumps between songs (it boosts quieter tracks more than it reins in loud ones) and applies across Original, Drum Boost, and Drumless.
  • Rendering — choose a custom renders folder (or reset to default) and set the render quality (bitrate) for new renders.
  • Diagnostics — Write debug log captures the app's system log to debug.log while enabled; use Reveal in Finder to share it.

Install

Backline Boost is currently a local macOS prototype built from source. There is not yet a signed, notarized installer package.

Requirements

  • macOS 14 or newer.
  • Xcode or Apple Command Line Tools with Swift 6 support.
  • No other tools are required — the drum-separation model ships inside the app, and Backline Boost analyzes, separates, and renders entirely on-device with no network access.

Build and launch

./script/build_and_run.sh run

For normal local use, double-click Launch Backline Boost.command in the repo.

Troubleshooting

Separation model isn’t ready

The on-device separation model ships inside the app, so it is ready as soon as Backline Boost is installed — nothing to download. The first render after installing (or updating) prepares the model, which can add a few seconds to that one render. If a render fails, use Retry render. Original playback always works.

Drum Boost or Drumless plays Original

The separated assets are not ready for that track yet. Wait for its status to reach Ready, or use Retry render if it failed. Backline Boost falls back to Original so playback remains usable.

macOS asks for permission on a Music drag

The first Apple Music drag shows a one-time system prompt asking to access Apple Music and your media library. Approve it — Backline Boost reads the dragged track’s file from your Music library and fetches its album artwork from Music’s database (Apple Music keeps artwork there, not inside the audio file). If you declined, grant it later in System Settings ▸ Privacy & Security ▸ Media & Apple Music.

Apple Music drag does nothing

Dragging a track directly from the Apple Music app works for local, purchased, or DRM-free downloaded songs that have a real audio file behind them. A protected Apple Music download (.m4p) is DRM-encrypted — Backline Boost can’t decode or separate it, so dropping one shows a “Can’t import this track” message. A subscription item that isn’t downloaded has no local file at all. Either way, use a purchased or file copy of the song, or drag it in from Finder.
